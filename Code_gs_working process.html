इमेज अपलोड और URL जनरेशन

डैशबोर्ड पर वापस जाएं
मौजूदा इमेज URL प्रक्रिया

जब कोई उपयोगकर्ता फोटो अपलोड करता है, तो वह फोटो सीधे Google स्प्रेडशीट में सेव नहीं होती है। इसके बजाय, फोटो को GitHub पर अपलोड किया जाता है, और केवल उस फोटो का लिंक (URL) स्प्रेडशीट में सेव होता है।

संबंधित फ़ाइल और फ़ंक्शन

यह पूरी प्रक्रिया Code.gs फ़ाइल के अंदर uploadImageToGitHub नामक फ़ंक्शन द्वारा नियंत्रित की जाती है।

प्रक्रिया के चरण (Step-by-Step Process)
  1. GitHub से कनेक्शन: स्क्रिप्ट आपके GitHub खाते से जुड़ती है।
  2. यूनिक फ़ाइल पाथ बनाना: हर इमेज के लिए एक यूनिक पाथ बनाया जाता है (जैसे: /school_images/School_Name/timestamp_filename.jpg)।
  3. GitHub API पर अपलोड: स्क्रिप्ट इमेज को GitHub पर भेजती है।
  4. URL प्राप्त करना: सफल अपलोड पर, GitHub एक डायरेक्ट डाउनलोड URL (download_url) देता है।
  5. स्प्रेडशीट में सेव करना: यही URL स्प्रेडशीट के PhotoURL कॉलम में सेव हो जाता है।
नई कस्टम URL प्रक्रिया (New Custom URL Process)

आपके अनुरोध के अनुसार, Code.gs को अब एक कस्टम इमेज URL बनाने के लिए संशोधित कर दिया गया है। अब GitHub द्वारा दिया गया URL उपयोग नहीं किया जाएगा।

नई प्रक्रिया कैसे काम करती है:
  1. इमेज हमेशा की तरह आपके GitHub रिपॉजिटरी में अपलोड होती है।
  2. अपलोड सफल होने के बाद, स्क्रिप्ट GitHub से मिले URL को अनदेखा कर देती है।
  3. इसके बजाय, यह "https://image.digitalerp.shop" डोमेन का उपयोग करके एक नया URL बनाती है।
  4. यह नया URL (जैसे: https://image.digitalerp.shop/school_images/School_Name/image.jpg) स्प्रेडशीट में सहेजा जाता है।

महत्वपूर्ण सेटअप!

यह URL तभी काम करेगा जब आप अपने डोमेन image.digitalerp.shop को सही ढंग से कॉन्फ़िगर करेंगे ताकि वह आपकी GitHub रिपॉजिटरी की रॉ फाइलों को दिखाए। इसके लिए आपको अपने डोमेन प्रोवाइडर पर CNAME या प्रॉक्सी सेटअप करना होगा।
