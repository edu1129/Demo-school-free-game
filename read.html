प्रोजेक्ट की संरचना और कार्य

Google Apps Script (GAS) फाइलें और उनके URL

हमारे सिस्टम में, अलग-अलग कामों के लिए अलग-अलग Google Apps Script फाइलें और उनके URL हैं। सर्वर (app.js) यह तय करता है कि कौन सी रिक्वेस्ट किस URL पर भेजनी है।

1. Code.gs (मुख्य स्क्रिप्ट)
  • URL मैपिंग: GAS_URL (सर्वर पर) -> /api (API रास्ता)
  • काम: यह स्क्रिप्ट प्रोजेक्ट के लगभग सभी मुख्य काम करती है, जैसे:
    • स्कूल रजिस्ट्रेशन
    • प्रिंसिपल और टीचर लॉगिन
    • छात्र और स्टाफ को जोड़ना/देखना
    • डैशबोर्ड के लिए डेटा लाना
    • फीस, सैलरी, और खर्चों का प्रबंधन करना
    • उपस्थिति (Attendance) रिकॉर्ड करना
2. Code2.gs (बकाया फीस स्क्रिप्ट)
  • URL मैपिंग: GAS_URL2 (सर्वर पर) -> /api2 (API रास्ता)
  • काम: यह स्क्रिप्ट विशेष रूप से उन छात्रों की सूची लाने के लिए है जिनकी फीस बकाया (Dues) है।
3. Code3.gs (डेटा व्यूअर स्क्रिप्ट)
  • URL मैपिंग: GAS_URL3 (सर्वर पर) -> /api3 (API रास्ता)
  • काम: यह एक एडमिन टूल है जो किसी स्कूल की स्प्रेडशीट की सभी शीट्स का पूरा डेटा एक साथ दिखाता है।
4. Code4.gs (परिणाम स्क्रिप्ट)
  • URL मैपिंग: GAS_URL4 (सर्वर पर) -> /api4 (API रास्ता)
  • काम: यह स्क्रिप्ट छात्रों को सार्वजनिक रूप से अपना परिणाम (Result) देखने की अनुमति देती है। इसके लिए लॉगिन की आवश्यकता नहीं है।
5. Code5.gs (पासवर्ड रीसेट स्क्रिप्ट)
  • URL मैपिंग: GAS_URL5 (सर्वर पर) -> /api5 (API रास्ता)
  • काम: यह स्क्रिप्ट "पासवर्ड भूल गए" (Forgot Password) की प्रक्रिया को संभालती है, जिसमें OTP भेजना और पासवर्ड रीसेट करना शामिल है।
HTML फाइलें और उनके काम

हर HTML फाइल एक खास पेज या फंक्शन का प्रतिनिधित्व करती है।

1. index.html
  • काम: यह मुख्य एप्लीकेशन पेज है। यहीं पर उपयोगकर्ता लॉगिन, रजिस्ट्रेशन और लॉगिन के बाद प्रिंसिपल या टीचर का डैशबोर्ड देखते हैं।
  • API उपयोग: यह मुख्य रूप से /api (यानी Code.gs) का उपयोग करता है।
2. list.html
  • काम: यह पेज बकाया फीस वाले छात्रों की सूची दिखाता है।
  • API उपयोग: यह /api2 (यानी Code2.gs) का उपयोग करता है।
3. data.html
  • काम: यह पेज प्रिंसिपल को उनकी पूरी स्कूल स्प्रेडशीट का डेटा देखने की अनुमति देता है।
  • API उपयोग: यह /api3 (यानी Code3.gs) का उपयोग करता है।
4. results.html
  • काम: यह एक सार्वजनिक पेज है जहां कोई भी छात्र अपना स्कूल कोड, कक्षा और रोल नंबर डालकर अपना परिणाम देख सकता है।
  • API उपयोग: यह /api4 (यानी Code4.gs) का उपयोग करता है।
5. password.html
  • काम: यह पेज प्रिंसिपल को अपना पासवर्ड रीसेट करने की सुविधा देता है।
  • API उपयोग: यह /api5 (यानी Code5.gs) का उपयोग करता है।
6. view.html
  • काम: यह एक साधारण पेज है जो ब्राउज़र में संग्रहीत कुकीज़ (cookies) को देखने के लिए है। यह किसी भी बैकएंड API का उपयोग नहीं करता है।
